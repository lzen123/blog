# FastAPI一个用于构建API的现代、快速（高性能）的Web框架

## 📖 运行一个FastAPI服务的两种方式

### 方式一：使用启动脚本`uv run python start_server.py`

```bash
# start_server.py
if __name__ == "__main__":
	uvicorn.run(
	    "api:app",           # 应用路径
	    host="0.0.0.0",      # 绑定地址
	    port=8000,           # 端口号
	    reload=True,         # 自动重载
	    log_level="info"     # 日志级别
	)
```

#### 语法解释

Python **入口点保护模式**：`if __name__ == "__main__`

```python
# start_server.py

# 场景 1: 直接运行脚本
python start_server.py
# 此时 __name__ = "__main__"
# ✅ 条件为 True，执行下方代码

# 场景 2: 作为模块导入
from start_server import something
# 此时 __name__ = "start_server"
# ❌ 条件为 False，不执行启动代码
```

**作用：**

- ✅ 防止导入时意外执行启动代码
- ✅ 确保只有直接运行时才启动服务器
- ✅ 符合 Python 最佳实践规范

### 方式二：直接使用 uvicorn

```bash
uvicorn api:app --reload --host 0.0.0.0 --port 8000
```

#### 参数解析

```
┌──────────────────────────────────────────────────────────┐
│  uvicorn api:app --reload --host 0.0.0.0 --port 8000    │
│  ─────── ───┬─── ─────── ─────── ─────── ─────── ────── │
│    │        │       │            │              │       │
│    │        │       │            │              └─ 端口  │
│    │        │       │            │
│    │        │       │            └───────── 主机          │
│    │        │       └─ 自动重载（开发模式）                │
│    │        └─ FastAPI 模块路径/应用对象                   │
│    └─ ASGI 服务器名称                                     │
└──────────────────────────────────────────────────────────┘
```

#### 参数说明

| 参数        | 值          | 含义     | 说明                                     |
| ----------- | ----------- | -------- | ---------------------------------------- |
| `app`       | `"api:app"` | 应用对象 | `api.py`文件中的 `app` 变量              |
| `host`      | `"0.0.0.0"` | 监听地址 | `0.0.0.0` 表示接受所有网络接口的连接     |
| `port`      | `8000`      | 端口号   | HTTP 服务端口，可自定义（如 8001, 8080） |
| `reload`    | `True`      | 自动重载 | 代码改变时自动重启（仅开发环境）         |
| `log_level` | `"info"`    | 日志级别 | debug/info/warning/error/critical        |
|             |             |          |                                          |

### 3. **`reload=True` 热重载机制**

```
┌─────────────────────────────────────────────────────────┐
│              热重载工作原理                              │
└─────────────────────────────────────────────────────────┘

┌──────────────┐
│  代码文件     │
│  api.py      │
└──────┬───────┘
       │ 修改并保存
       ▼
┌─────────────────────────────────────────┐
│  Watchdog 进程监控文件变化               │
│  - 检测到 .py 文件改变                    │
│  - 触发重新加载                          │
└──────┬──────────────────────────────────┘
       │
       ▼
┌─────────────────────────────────────────┐
│  主进程优雅重启                          │
│  1. 等待当前请求完成                     │
│  2. 重新加载代码                         │
│  3. 启动新进程                          │
│  4. 转移连接                            │
└──────┬──────────────────────────────────┘
       │
       ▼
┌──────────────┐
│  无需手动重启 │
│  立即生效！   │
└──────────────┘
```

**注意事项：**

- ⚠️ `reload=True` 仅用于开发环境
- ⚠️ 生产环境请使用 `reload=False`
- ⚠️ 热重载会增加 CPU 和内存消耗

### 事件循环&并发处理流程

```
┌──────────────────────────────────────────────────────────────┐
│              Uvicorn 服务器接收并发连接                       │
└──────────────────────────────────────────────────────────────┘

时刻 t0:
客户端 1 ──→ HTTP 请求 1 ──┐
                          │
客户端 2 ──→ HTTP 请求 2 ──┼──→ Uvicorn 接收
                          │
客户端 3 ──→ HTTP 请求 3 ──┘
                          │
                          ▼
              ┌───────────────────────┐
              │   Event Loop (while True) │
              │  (单线程，事件驱动)    │
              └───────────┬───────────┘
                          │
        ┌─────────────────┼─────────────────┐
        │                 │                 │
        ▼                 ▼                 ▼
┌───────────────┐ ┌───────────────┐ ┌───────────────┐
│   请求 1 处理   │ │   请求 2 处理   │ │   请求 3 处理   │
│               │ │               │ │               │
│ async def     │ │ async def     │ │ async def     │
│   await db    │ │   await api   │ │   await file  │
│   ⏸️ 暂停      │ │   ⏸️ 暂停      │ │   ⏸️ 暂停      │
└───────┬───────┘ └───────┬───────┘ └───────┬───────┘
        │                 │                 │
        └─────────────────┴─────────────────┘
                          │
                          ▼
              事件循环调度器：
              "请求 1 在等数据库，先处理请求 2"
              "请求 2 在等 API，先处理请求 3"
              "请求 3 完成了，返回结果"
              "请求 2 的 API 好了，继续"
              ...
```

### 异步并发模型

```python
# FastAPI 基于 async/await 的异步模型

@app.post("/items/")
async def create_item(item: ItemCreate):
    """异步端点函数"""
    # 1. 非阻塞 I/O 操作
    result = await some_async_operation()
    return result

# 2. 事件循环可以处理其他请求
#    而不会阻塞当前线程
```

### ASGI 协议工作流程

```
客户端请求 → ASGI Scope → FastAPI 处理 → ASGI Send → 客户端响应

详细流程：

1. 接收请求
   ┌─────────────────────────────────────────┐
   │  ASGI Scope (字典)                      │
   │  {                                      │
   │    "type": "http",                      │
   │    "method": "POST",                    │
   │    "path": "/items/",                   │
   │    "headers": [...],                    │
   │    "query_string": b""                  │
   │  }                                      │
   └─────────────────────────────────────────┘

2. 调用应用
   ┌─────────────────────────────────────────┐
   │  await app(scope, receive, send)        │
   │  - scope: 请求信息                       │
   │  - receive: 接收请求体                   │
   │  - send: 发送响应                        │
   └─────────────────────────────────────────┘

3. FastAPI 处理
   ┌─────────────────────────────────────────┐
   │  - 路由匹配                              │
   │  - 依赖注入                              │
   │  - 数据验证 (Pydantic)                   │
   │  - 执行端点函数                          │
   │  - 生成响应                              │
   └─────────────────────────────────────────┘

4. 返回响应
   ┌─────────────────────────────────────────┐
   │  ASGI Send                              │
   │  {                                      │
   │    "type": "http.response.start",       │
   │    "status": 201,                       │
   │    "headers": [...]                     │
   │  }                                      │
   │  {                                      │
   │    "type": "http.response.body",        │
   │    "body": b"{\"id\": ...}"             │
   │  }                                      │
   └─────────────────────────────────────────┘
```

## 📊 性能对比

| 运行方式                        | 启动速度   | 热重载 | 适用场景         |
| ------------------------------- | ---------- | ------ | ---------------- |
| `uv run python start_server.py` | ⭐⭐⭐     | ✅     | 开发环境（推荐） |
| `uvicorn api:app --reload`      | ⭐⭐⭐     | ✅     | 开发环境（快捷） |
| `uvicorn api:app`               | ⭐⭐⭐⭐⭐ | ❌     | 生产环境         |
| `gunicorn + uvicorn`            | ⭐⭐⭐⭐   | ❌     | 高并发生产环境   |

### 生产环境

```bash
# 关闭热重载
uvicorn api:app --host 0.0.0.0 --port 8000 --workers 4

# 或使用 gunicorn（需要安装）
gunicorn api:app \
  --bind 0.0.0.0:8000 \
  --workers 4 \
  --worker-class uvicorn.workers.UvicornWorker
```

这就是完整的FastAPI运行方式和执行原理解析！😊
